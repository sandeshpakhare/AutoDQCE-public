# Future Roadmap

This document outlines **planned and potential future enhancements** for AutoDQCE.

Items listed here represent direction, not commitment.

---

## Short-Term Enhancements

- Expanded documentation and diagrams
- Improved validation summaries
- Additional example datasets
- Clearer configuration documentation

---

## Medium-Term Goals

- Configuration-driven rule definition examples
- Pluggable validation strategies
- Dataset comparison reports (before vs after)
- Quality scoring improvements

---

## Long-Term Vision

- Integration hooks for ML pipelines
- CLI-based execution interface
- Support for additional structured formats
- Optional open-sourcing of non-core utilities

---

## Non-Goals

AutoDQCE intentionally avoids:
- Becoming a full ETL framework
- Replacing domain-specific validation logic
- Performing black-box automatic fixes

The focus remains on **data quality and correctness**, not general data processing.
